# JobSwiper 🔥

Tinder для вакансий с hh.ru. Свайпайте вакансии, узнавайте ИИ-совместимость со
своим резюме и получайте готовое сопроводительное письмо под каждую вакансию.

## Возможности

- **Лента-свайпер** — карточки вакансий с hh.ru, свайп вправо (отклик) / влево (мимо),
  кнопки и стрелки клавиатуры.
- **ИИ-совместимость** — для каждой вакансии модель `mistral-nemo` (Mistral через AITunnel)
  оценивает соответствие резюме в процентах, с плюсами и пробелами. Считается батчами
  и лениво, чтобы укладываться в бесплатный лимит.
- **Личный кабинет** — загрузите резюме файлом **PDF или DOCX** (текст извлекается
  на сервере) либо вставьте текст вручную. ИИ определит профессию, навыки и уровень
  и подставит профессию в фильтр поиска.
- **Отклики** — лайкнутые вакансии собираются в отдельной вкладке. Для каждой ИИ пишет
  персональное сопроводительное письмо: его можно скопировать в один клик и перейти
  по кнопке «Откликнуться на hh.ru».
- **Письмо под свою вакансию** — на вкладке «Отклики» можно вставить текст любой
  вакансии (не из hh.ru) — ИИ напишет под неё сопроводительное письмо, которое
  сохранится в списке откликов с пометкой «Своя вакансия».
- **Фильтры** — профессия, город, опыт, график, занятость, зарплата, сортировка.
- **Авторизация** — вход через Google OAuth в Supabase Auth; после входа данные
  синхронизируются между устройствами.
- Тёмная/светлая тема, адаптивный мобильный дизайн.

## Стек

Next.js 16 (App Router) · TypeScript · Tailwind v4 · shadcn/ui (Base UI) ·
Framer Motion · Zustand · TanStack Query · Supabase (Postgres) · unpdf + mammoth
(извлечение текста из PDF/DOCX).

## Запуск

1. Ключи уже лежат в `.env.local`:
   ```
   AITUNNEL_API_KEY=...
   HH_CLIENT_ID=...
   HH_CLIENT_SECRET=...
   SUPABASE_URL=...                 # проект Supabase
   SUPABASE_PUBLISHABLE_KEY=...     # publishable-ключ (используется только на сервере)
   NEXT_PUBLIC_SITE_URL=https://jobswiper.ru  # production URL для OAuth callback
   ```
   В Google OAuth должен быть разрешён Supabase callback
   `https://<project-ref>.supabase.co/auth/v1/callback`, а в Supabase Auth Redirect
   URLs — `http://localhost:3000/auth/callback` для dev и
   `https://jobswiper.ru/auth/callback` для production.
2. Установка и старт:
   ```bash
   npm install
   npm run dev
   ```
3. Откройте http://localhost:3000

## Как это устроено

- Ключи API используются **только на сервере** (в `/app/api/*`), в браузер не попадают.
- Поиск вакансий hh.ru требует токен приложения (анонимные запросы ловит анти-бот).
  Токен получается через `client_credentials` и кэшируется в Supabase (таблица
  `app_tokens`, см. `src/lib/hh/token.ts`), автоматически обновляясь по истечении.
- Резюме, вакансии, оценки соответствия, отклики и письма хранятся в **Supabase
  (Postgres)**. Для гостей используется анонимный `userId` устройства; после входа
  через Google API-роуты используют verified `auth.getUser().id`. Доступ к БД —
  только через серверные `/api/db/*` роуты (ключ Supabase не попадает в браузер).

## Структура

```
src/
  app/
    page.tsx              лента-свайпер
    liked/page.tsx        отклики + письма
    profile/page.tsx      личный кабинет
    api/                  прокси к hh.ru и AITunnel
  components/             карточки, колода, фильтры, диалог вакансии…
  lib/
    hh/                   клиент hh.ru, токен, форматирование, справочники
    ai/                   клиент AITunnel (Mistral) + промпты
    store/                Zustand-стор
    hooks/                загрузка вакансий, расчёт совместимости
```
